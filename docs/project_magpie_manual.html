Project Magpie Manual

현재 코드 구조를 빠르게 다시 잡는 운영형 안내서

이 문서는 예전 기획 문서가 아니라, 지금 저장소에 실제로 연결된 엔트리포인트와 그래프 흐름을 기준으로 정리한 매뉴얼입니다. 사용자 기본 그래프, Bat Daemon, Backtest, Dashboard, MongoDB 저장 구조를 짧고 선명하게 다시 파악하는 데 초점을 맞춥니다.

지금 가장 중요한 요약

  • 사용자 기본 그래프는 Owl -> Fox -> Parallel Coordinator -> Hawk입니다.
  • 코인별 분석은 Coordinator 내부의 per_coin_pipeline에서 병렬로 수행됩니다.
  • 실시간 거래 실행 주체는 Agent가 아니라 bat_daemon/run.py입니다.
  • Target Refresh Graph는 SELL 후 EXPIRED 타점을 새로 계산합니다.
  • Signal Trigger, Daily Report는 현재 미연결/확장 예정 그래프입니다.

  • magpie_agent/run.py: Telegram 기반 사용자 진입점

실시간 규칙

  • WAITING_BUY, HOLDING만 감시합니다.
  • TOUCH BUY와 CLOSE BUY를 모두 검사합니다.
  • SELL은 익절가 또는 손절가 조건으로 발생합니다.
  • 체결은 Agent가 아니라 Daemon이 직접 수행합니다.

체결 이후

  • BUY는 지갑 KRW 잔고에 buy_allocation_pct를 곱해 수량을 계산합니다.
  • SELL은 해당 자산 보유량 전량 기준입니다.
  • SELL 완료 후 타점 상태는 EXPIRED로 바뀝니다.
  • 이후 refresh 그래프가 새 WAITING_BUY 타점을 계산합니다.

MongoDB 저장 구조

컬렉션 주요 작성 주체 핵심 내용
strategies Owl, Hawk 전략 본문과 최종 target_coins
monitoring_targets Calculate Team, Hawk 정리 로직 매수 범위, 익절/손절, 트리거 기준, buy_allocation_pct, 상태
wallets Daemon, wallet helper KRW 잔고, 보유 자산, 전체 trade_history

monitoring_targets에서 기억할 것

  • Daemon은 현재 WAITING_BUY, HOLDING만 감시합니다.
  • EXPIRED는 감시 대상에서 빠지고 refresh 대상이 됩니다.
  • Hawk는 최종 미선정 코인의 타점을 지울 수 있습니다.

wallets에서 기억할 것

  • 잔고만 있는 단순 지갑이 아닙니다.
  • 코인별 보유 수량과 평균 단가가 들어갑니다.
  • trade_history가 저장되어 다음 계산 컨텍스트에도 활용됩니다.
